# Snow Foundation Demo Script

Date: 2026-04-12
Status: Internal
Goal: 30-minute working session for Snow Foundation

## Demo principle

Do not demo raw tools.

Do not start with feature lists.

Do not open on a dashboard.

Start by showing that we understand Snow's own language, then move from portfolio legibility to project evidence.

## Opening frame

Say this in substance:

"We took Snow's public strategy seriously and used it as the frame for a working portfolio view. The question we wanted to test was whether Snow's portfolio can be made more legible across capital, place, and community evidence in one chain."

## Segment 1 — Snow's stated strategy

Time: 5 minutes

Anchor the conversation in Snow's own public material:

- Purpose & Approach
- Pillars
- 2024 Annual Report
- RHD strategy statement

Key points to say:

- Snow already speaks in terms of place, partnership, shared intelligence, and community-led work.
- The platform is not imposing a foreign logic onto Snow.
- It is trying to make Snow's own logic easier to inspect and act on.

## Segment 2 — CivicGraph / GrantScope

Time: 10 minutes

Show the Snow portfolio as it currently appears after the 2024 repair.

Lead with:

- annual giving baseline now stored: `$13,704,202`
- `68` grantee rows in refreshed foundation views
- `58` linked grantees across `5` states
- `$4,923,622` in currently surfaced grant value
- `44` verified amount-backed 2024 relationships in the graph
- `$4,823,622` in verified 2024 grant value
- `13` Snow program surfaces, `12` currently open-like
- readiness tier: `high`

Then show three things only:

### A. Portfolio concentration

- social justice
- Canberra flagships
- South Coast
- Sydney
- social enterprise / innovation

### B. Example recipients

- FRRR
- NACCHO
- Project Independence
- EveryMan Australia
- Tender Funerals Australia
- Orange Sky Australia
- Equality Australia

### C. Decision value

Explain that this is useful because it makes it easier to prepare:

- a leadership memo
- a partner brief
- a place-based review
- a funding opportunity scan

## Segment 3 — Empathy Ledger

Time: 8 minutes

Only show the strongest safe layer.

Open with:

- Deadly Hearts Trek is a live Snow project
- the project framing is now aligned to the RHD work
- the tenant currently has `3` Snow-linked transcripts for this project
- the broader analysis layer already shows `120` impact insights across `8` analyzed transcripts in the repo-level workflow

Then show:

### A. Project framing

- early diagnosis
- community education
- reduced deaths

### B. Cultural method

- two-way learning
- elder-led briefings
- community invitation
- cultural safety

### C. Why this matters

- it keeps the evidence layer close to community voice
- it makes Snow's RHD work more legible as lived practice, not only funding

## Segment 4 — Combined value

Time: 4 minutes

Say this plainly:

"The value is not GrantScope by itself or Empathy Ledger by itself. The value is one evidence chain:

- strategy
- capital
- recipient network
- geography
- project evidence
- narrative"

That is the point where Snow should feel the difference.

## Close

Time: 3 minutes

Ask for one of two pilots:

### Option 1

RHD and social determinants portfolio pilot

### Option 2

ACT / NSW South Coast place-based portfolio pilot

## What not to show

- raw tenant stories list
- archived smoke-test story artifacts
- repo prospectus claims as if they are current live metrics
- any unverified member or partner record
- exploratory clicking through unfinished routes

## Current caveats to disclose if needed

- all `44` verified 2024 annual-report rows are now entity-matched into the graph
- the broader Snow portfolio beyond that verified 2024 layer is still incomplete and should be presented as public-surface intelligence, not full historical portfolio truth
- the Snow member layer should still not be presented as contact-grade identity data because `7` of `8` active members now have blank/null email values after placeholder cleanup
- the platform is ready for a curated walkthrough, not an open-ended raw demo
